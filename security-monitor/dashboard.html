🔒 Security Monitor

📖 About This Monitor

🔍 What Is This?

This is a personal PC security monitoring agent built specifically for Den's Windows machine. It watches your computer for signs of malware, unauthorized access, and data loss — just like an enterprise security system, but running locally on your own PC.

🎯 What It Does

The agent continuously monitors four areas of your system:

  • 🔄 Running Processes — It keeps a list of every program that runs on your PC. When a new, unknown process appears that wasn't seen during the learning phase, it flags it. This catches malware, coin miners, backdoors, and suspicious tools.
  • 🌐 Network Connections — It tracks outbound connections to external IPs. When your PC connects to an IP address it has never connected to before, it raises an alert. This catches command-and-control (C2) callbacks, data exfiltration, and unauthorized phone-homes.
  • 📁 Suspicious Files — It watches sensitive folders (Downloads, Desktop, Temp) for new executable files (.exe, .dll, .ps1, .bat, .vbs, .js, .scr, .jar). These are common malware delivery formats. When one appears, it alerts you.
  • 📋 Security Events — It reads the Windows Security Event Log for failed login attempts (Event ID 4625) and other critical security events. Multiple failed logins can indicate a brute-force attack or someone trying to break into your PC.

🧠 How It Works (Two Phases)

Phase 1: Learning 🟡

  • When first launched, the agent enters learning mode.
  • It runs every 15 minutes via a cron job and records everything it sees — every running process, every network connection, every file, every security event.
  • This builds a "baseline" database of what's normal for your PC.
  • During learning, no real alerts are sent — it only shows you a sample alert so you know what to expect.
  • You can monitor progress on the dashboard. The more data collected, the better the baseline.

Phase 2: Active Monitoring 🟢

  • When you click "START MONITORING" on the dashboard (or tell me to start), the agent switches to active monitoring mode.
  • Now, anything not in the baseline triggers a real alert sent directly to you on Telegram.
  • The alert includes: what was detected, severity level, timestamp, and details.
  • New processes and files are high-severity. New network connections are medium-severity.

🛠️ How It Was Built

This agent was built using Python running on Den's Windows PC, orchestrated by Hermes Agent (the AI assistant you're talking to). Here's the technical breakdown:

  • Scanner Script (security_agent.py): A Python script that runs every 15 minutes via Hermes cron. It uses:
    • tasklist (Windows CLI) to enumerate running processes
    • netstat -ano to list active network connections
    • os.listdir() + extension filtering to detect suspicious files
    • wevtutil (Windows Event Log CLI) to query security events
  • Database: SQLite stores the baseline of known-good processes, connections, files, and events. It also stores all alerts and activity logs.
  • Dashboard (dashboard_server.py + dashboard.html): A lightweight Python HTTP server that serves a real-time web dashboard. It exposes JSON API endpoints that the HTML page fetches every 5 seconds for live updates.
  • Cron: Hermes' built-in cron scheduler runs the scanner every 15 minutes. During learning, it silently accumulates data. During monitoring, it delivers alerts to Telegram.
  • Delivery: Alerts are printed to stdout by the scanner script. The cron job captures stdout and delivers it directly to Den's Telegram chat.

📊 Data Stored

Everything is stored locally in C:\Users\decni\AppData\Local\hermes\security-agent\baseline.db — never sent anywhere else.

⚠️ Limitations

  • This is a behavioral baseline system, not a traditional antivirus. It detects things that are new, not things that are known-bad.
  • It cannot scan inside archives, decrypt traffic, or perform deep packet inspection.
  • False positives are possible — for example, a legit software update might appear as a new process.
  • It only runs when your PC is on and Hermes is active.
